# RSA 非对成加密解密

## 1、解决需求
- 当诸如 身份证号、银行卡号 直接存入数据库就有泄漏的风险，所以需要非对称加密后存储。但非对成加密后的密文无法用于数据库查询，所以需要在数据表中增加两个字段，1密文字段、2散列字段，从而达到字符串非对成加密后，依然可以进行数据库查询。

## 2、测试
1. ` ./test.php ` 测试代码
2. ` ./create_keys.php ` 生成公钥、密钥

## 3、备注
1. 散列盐为：` $=$.salt.$=$ `
2. 公钥文件：` ./src/public.key `
3. 密钥文件：` ./src/private.key `
